# 사주/점성술 AI 챗봇 앱 구조 설계

## 1. 추천 기술 스택

| 영역 | 추천 | 이유 |
|---|---|---|
| 앱 프레임워크 | React Native (Expo) | iOS/Android 동시 빌드, 초보자 접근성 좋음 |
| 백엔드 | Supabase | Auth + Postgres + Edge Functions 올인원, 서버 관리 불필요 |
| AI 대화 | Claude API (claude-sonnet-5) | 페르소나 프롬프트 기반 해석/대화 |
| 사주 계산 | @fullstackfamily/manseryeok (npm) | 진태양시 보정까지 지원하는 한국 만세력 라이브러리 |
| 음력/간지 계산 | lunar-javascript (npm) | 八字, 간지, 절기 등 폭넓게 지원, 자미두수 기초 데이터에도 활용 가능 |
| 서양 점성술 | astronomy-engine (npm) 또는 외부 천문 API | 행성 위치 계산 (직접 만들지 말 것) |
| 결제/구독 | RevenueCat | 인앱결제(이용권 차감형) 관리 대행 |
| 참고 오픈소스 | github.com/rath/orrery | 사주팔자·자미두수·서양 출생차트를 브라우저에서 계산하는 예제. 라이선스 확인 후 로직 참고용으로만 활용 |

## 2. 화면 흐름 (Screen Flow)

```
스플래시
  └▶ 온보딩 (생년월일시 / 출생지 / 성별 / 양·음력 선택)
        └▶ 계산 로딩 화면 (사주/점성술/자미두수 계산 중 애니메이션)
              └▶ 분석 대시보드
                    ├─ 사주팔자 요약 카드
                    ├─ 점성술 네이탈 요약 카드
                    ├─ 자미두수 명반 요약 카드
                    └▶ AI 챗 화면 (메인)
                          ├─ 카테고리 버튼: 연애운 / 재물운 / 직업운 / 오늘의 운세
                          ├─ 대화 히스토리
                          └─ 이용권 소진 시 → 결제 모달
                    └▶ 마이페이지
                          ├─ 저장한 분석 결과
                          ├─ 이용권 잔여 횟수
                          └─ 대화 기록 다시보기
```

## 3. 데이터 흐름

1. 사용자가 생년월일시/출생지 입력
2. 클라이언트 또는 Supabase Edge Function에서 `manseryeok`, `lunar-javascript`, 천문 계산 라이브러리로 사주/점성술/자미두수 원본 데이터(간지, 행성 위치, 명반 등) 계산
3. 계산된 구조화 데이터(JSON)를 Claude API 시스템 프롬프트에 컨텍스트로 삽입
4. Claude가 페르소나("호리") 톤으로 해석 및 대화 응답 생성
5. 대화 기록은 Supabase DB에 저장 (재방문 시 컨텍스트 유지)

## 4. 핵심 설계 원칙

- **계산과 해석의 분리**: 간지/행성 위치 같은 정확한 계산은 절대 AI에게 맡기지 않는다. 라이브러리로 계산한 뒤, 그 결과만 AI에게 넘겨 해석시킨다.
- **이용권 UX**: 스크린샷처럼 "질문 1회 = 답변 1회" 차감형 구조를 쓸 경우, 사용자가 남은 횟수를 항상 볼 수 있게 하는 것이 클레임을 줄인다.
- **면책 문구**: 사주/점성술은 오락·자기이해 목적임을 온보딩이나 첫 답변에 명시하는 앱이 많다 (분쟁 예방 차원).
